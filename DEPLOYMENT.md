# Lonaat Backend - Deployment Guide

## Overview
This guide covers deploying the Lonaat backend to Render.com, a modern cloud platform that's free-tier friendly and easy to use.

---

## Prerequisites

Before deploying, ensure you have:
1. ✅ A GitHub account
2. ✅ A Render.com account (sign up at https://render.com)
3. ✅ All environment secrets ready:
   - `ADMIN_USERNAME`
   - `ADMIN_PASSWORD`
   - `ENCRYPTION_KEY`
   - `FIREBASE_SERVICE_ACCOUNT` (JSON string)
   - `FLASK_SECRET`

---

## Deployment Steps

### Step 1: Push Code to GitHub

```bash
# Initialize git repository (if not already done)
git init

# Add all files
git add .

# Commit changes
git commit -m "Initial commit - Lonaat Backend with encryption"

# Add your GitHub repository as remote
git remote add origin https://github.com/YOUR_USERNAME/lonaat-backend.git

# Push to GitHub
git push -u origin main
```

### Step 2: Connect Render to GitHub

1. **Go to Render Dashboard**: https://dashboard.render.com
2. **Click "New +"** → Select **"Web Service"**
3. **Connect your GitHub repository**:
   - Click "Connect Account" if first time
   - Search for `lonaat-backend` repository
   - Click "Connect"

### Step 3: Configure Web Service

Render will auto-detect the `render.yaml` file. Verify these settings:

**Basic Settings:**
- **Name**: `lonaat-backend`
- **Environment**: `Python 3`
- **Region**: Choose closest to your users
- **Branch**: `main`

**Build & Deploy:**
- **Build Command**: `pip install -r requirements.txt`
- **Start Command**: `gunicorn --bind 0.0.0.0:$PORT --workers 2 --timeout 120 main:app`

### Step 4: Add Environment Variables

In the Render dashboard, add these environment variables:

#### Required Secrets:

```bash
# Admin Credentials
ADMIN_USERNAME=your_admin_username
ADMIN_PASSWORD=your_strong_password

# Encryption Key (generate with command below)
ENCRYPTION_KEY=your_base64_encryption_key

# Flask Secret (auto-generated by Render)
FLASK_SECRET=auto_generated_or_custom

# Firebase Configuration
FIREBASE_DATABASE_URL=https://lonaat-93a89-default-rtdb.firebaseio.com
FIREBASE_SERVICE_ACCOUNT={"type":"service_account","project_id":"lonaat-93a89",...}
```

#### How to Generate ENCRYPTION_KEY:
```bash
python -c "import base64, os; print(base64.urlsafe_b64encode(os.urandom(32)).decode())"
```

**IMPORTANT**: 
- ✅ Mark all secrets as **"Secret"** in Render (won't be visible after saving)
- ✅ `FIREBASE_SERVICE_ACCOUNT` should be the **entire JSON** from Firebase Console
- ✅ Back up your `ENCRYPTION_KEY` securely - if lost, encrypted data cannot be recovered

### Step 5: Deploy

1. **Click "Create Web Service"**
2. Render will:
   - Clone your repository
   - Install dependencies from `requirements.txt`
   - Start the application with Gunicorn
3. **Wait for deployment** (usually 2-3 minutes)
4. **Your app will be live** at: `https://lonaat-backend-XXXX.onrender.com`

## Post-Deployment Verification

### Test Your Deployment

**1. Check Homepage:**
```bash
curl https://your-app.onrender.com/
```
Should return the Lonaat homepage HTML.

**2. Test Admin Login:**
```bash
curl -X POST https://your-app.onrender.com/admin_login \
  -H "Content-Type: application/json" \
  -d '{"username":"YOUR_ADMIN_USERNAME","password":"YOUR_ADMIN_PASSWORD"}'
```
Should return: `{"success": true}`

**3. Verify Firebase Connection:**
Check the Render logs for:
```
✅ Firebase initialized successfully! Database URL: https://lonaat-93a89-default-rtdb.firebaseio.com/
```

**4. Test Encrypted Payout:**
```bash
# Create a test payout
curl -X POST https://your-app.onrender.com/register_payout \
  -H "Content-Type: application/json" \
  -d '{
    "user_id":"test_user",
    "amount":5000,
    "bank_name":"Test Bank",
    "account_number":"1234567890",
    "account_name":"Test User"
  }'
```

Should return: `{"message":"payout requested","payout_id":"..."}`

## Troubleshooting

### Common Issues & Solutions

**Issue: "Address already in use"**
```
Solution: Render automatically assigns $PORT. 
Make sure start command uses: --bind 0.0.0.0:$PORT
```

**Issue: "ENCRYPTION_KEY is mandatory"**
```
Solution: Add ENCRYPTION_KEY to Render environment variables.
Generate with: python -c "import base64, os; print(base64.urlsafe_b64encode(os.urandom(32)).decode())"
```

**Issue: "Firebase initialization failed"**
```
Solution: Ensure FIREBASE_SERVICE_ACCOUNT contains valid JSON.
Check for proper escaping of quotes.
```

**Issue: "Module not found"**
```
Solution: Add missing package to requirements.txt
Deploy again to reinstall dependencies
```

**Issue: "Admin login fails"**
```
Solution: Verify ADMIN_USERNAME and ADMIN_PASSWORD are set correctly in Render.
Check for typos or extra spaces.
```

## Security Best Practices

### Production Security Checklist

✅ **Environment Variables:**
- All secrets stored in Render environment variables (not code)
- Marked as "Secret" to hide from logs
- Never committed to Git

✅ **Encryption:**
- `ENCRYPTION_KEY` backed up securely
- All payout bank details encrypted with AES-256-GCM

✅ **Firebase:**
- Database rules configured (not in test mode)
- Service account credentials secured

✅ **Admin Access:**
- Strong admin password (16+ characters, mixed case, numbers, symbols)
- Session-based authentication
- Audit logging enabled

✅ **HTTPS:**
- SSL certificate auto-provisioned by Render
- All traffic encrypted in transit

## Backup & Recovery

### Critical Data to Backup

**1. Environment Variables**
Save these securely (password manager recommended):
- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`
- `ENCRYPTION_KEY` ⚠️ **CRITICAL** - if lost, encrypted payouts cannot be recovered
- `FIREBASE_SERVICE_ACCOUNT`

**2. Firebase Database**
Firebase Realtime Database has automatic backups, but you can export:
```bash
# Export all data
curl https://lonaat-93a89-default-rtdb.firebaseio.com/.json > firebase_backup.json
```

**3. Code Repository**
Your GitHub repository serves as code backup.
