## 🔧 Common Issues & Fixes

### Issue 1: "Not Found" (404 Error)

**Possible Causes:**
1. ✅ **App is sleeping** (Free tier): First request takes 30-60 seconds to wake up
2. ❌ **App failed to start**: Check logs for errors
3. ❌ **Environment variables missing**: See Fix #2 below

**Solution:**
Wait 1 minute and try again. Free tier apps sleep after 15 minutes of inactivity.

### Issue 2: Missing Environment Variables ⚠️

**Most Common Issue!** Your app needs these environment variables:

**Go to:** Render Dashboard → Your Service → **Environment** tab

**Required Variables:**

| Variable | Where to Get It | Critical? |
|----------|-----------------|-----------|
| `ADMIN_USERNAME` | Your choice (e.g., "admin") | ✅ Yes |
| `ADMIN_PASSWORD` | Your choice (strong password) | ✅ Yes |
| `ENCRYPTION_KEY` | See `NEW_ENCRYPTION_KEY.txt` | ✅ Yes |
| `FIREBASE_SERVICE_ACCOUNT` | Firebase Console (JSON) | ✅ Yes |

| `FLASK_SECRET` | Auto-generated by Render | ✅ Yes |
| `OPENAI_API_KEY` | Optional (for AI features) | ❌ No |

1. Click **"Environment"** tab

### Issue 3: Build Failures

**Check Render Logs for:**

**Missing Python Package:**
```
ModuleNotFoundError: No module named 'firebase_admin'
```

**Fix:** Verify `requirements.txt` includes all packages:
```bash
# In Replit, check requirements.txt
cat requirements.txt
```

Should include:
```
flask
flask-cors
firebase-admin
openai
gunicorn
cryptography
python-dotenv
requests
tenacity
beautifulsoup4
```

**Python Version Mismatch:**
```
ERROR: Python 3.x is required
```

**Fix:** Render uses Python 3.11 by default (matches your setup)

### Issue 4: Firebase Connection Errors

**Error in Logs:**
```
Firebase initialization failed
Error: Invalid Firebase credentials
```

**Fix:**
1. Verify `FIREBASE_SERVICE_ACCOUNT` is the **complete JSON**
2. Check `FIREBASE_DATABASE_URL` matches:
   ```
   https://lonaat-93a89-default-rtdb.firebaseio.com
   ```
3. Ensure Firebase Realtime Database is enabled

### Issue 5: Gunicorn Startup Errors

**Error in Logs:**
```
Failed to find application object 'app' in 'main'
```

**Fix:** This should not happen. Your `render.yaml` correctly specifies:
```yaml
startCommand: gunicorn --bind 0.0.0.0:$PORT --workers 2 --timeout 120 main:app
```

And `main.py` has: `app = Flask(__name__)` ✅

## 💡 Free Tier Limitations

**Render Free Tier:**
- ✅ 750 hours/month (enough for 24/7)
- ⚠️ **Auto-sleep after 15 min inactivity**
- ⚠️ First request after sleep: 30-60 seconds to wake
- ⚠️ Slower CPU/memory than paid tiers

**If you need always-on:**
- Upgrade to Render Starter: $7/month
- No sleep, better performance

## 🆘 Still Not Working?

### Get Detailed Error Info

1. **Copy Render Logs:**
   - Logs tab → Copy last 50 lines
   - Look for ERROR or CRITICAL messages

2. **Check Firebase:**
   ```bash
   # Verify Firebase is accessible
   curl -X GET https://lonaat-93a89-default-rtdb.firebaseio.com/.json
   ```
   Should return: `{"error":"Permission denied"}`  
   (This is GOOD - means database exists but is protected)

3. **Check if port is binding:**
   - Logs should show: `Listening at: http://0.0.0.0:XXXXX`
   - If not, Gunicorn failed to start

### Common Log Errors

**Error:** `ModuleNotFoundError: No module named 'affiliate_scraper'`  
**Fix:** Ensure `affiliate_scraper.py` is in GitHub repo

**Error:** `ImportError: cannot import name 'AffiliateNetworkManager'`  
**Fix:** Ensure `affiliate_integration.py` is in GitHub repo

**Error:** `SyntaxError` in Python files  
**Fix:** Check Python syntax locally first
